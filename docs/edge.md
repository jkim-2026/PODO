# 구현 계획 - 엣지 추론 (Jetson)

이 계획서는 PCB 결함 탐지 시스템의 Jetson 엣지 추론 및 전송 구현 단계를 설명합니다.

## 진행 상황

| 항목 | 상태 |
|------|------|
| 프로젝트 구조 설계 | ⏳ 예정 |
| 모델 로딩 | ⏳ 예정 |
| 추론 파이프라인 | ⏳ 예정 |
| 백엔드 전송 | ⏳ 예정 |
| 에러 핸들링 | ⏳ 예정 |

## 프로젝트 구조

```
serving/edge/
├── main.py             # 메인 실행 파일
├── inference.py        # 모델 추론
├── sender.py           # 백엔드로 결과 전송
├── config/             # 설정
└── requirements.txt
```

## 백엔드 API 연동

**POST /detect 요청 형식:**

| 필드 | 타입 | 필수 | 설명 |
|------|------|------|------|
| timestamp | string | O | ISO 8601 형식 |
| image_id | string | O | 이미지 식별자 |
| result | string | O | "normal" 또는 "defect" |
| confidence | float | O | 신뢰도 (0.0~1.0) |
| defect_type | string | X | 결함 종류 (불량일 때만) |
| bbox | list[int] | X | [x1, y1, x2, y2] (불량일 때만) |
| image | string | X | Base64 인코딩 이미지 (불량일 때만) |

## 주요 기능

### 1. 모델 추론
- YOLO 모델 로딩
- 프레임별 추론
- 결과 후처리 (NMS 등)

### 2. 결과 전송
- 백엔드 API 호출
- 비동기 전송
- 전송 실패 시 처리

### 3. 이미지 처리
- 불량 이미지 Base64 인코딩
- bbox 좌표 추출

## 협의 사항

| 항목 | 결정 |
|------|------|
| 전송 주기 | 매 프레임 |
| 전송 방식 | 비동기 |
| 전송 실패 시 | 데이터 유실 (재전송 안 함) |
