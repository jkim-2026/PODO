# 구현 계획 - 프론트엔드 (Streamlit)

이 계획서는 PCB 결함 탐지 시스템의 Streamlit 대시보드 구현 단계를 설명합니다.

## 진행 상황

| 항목 | 상태 |
|------|------|
| 프로젝트 구조 설계 | ⏳ 예정 |
| 메인 대시보드 레이아웃 | ⏳ 예정 |
| 실시간 통계 표시 | ⏳ 예정 |
| 결함 이미지 갤러리 | ⏳ 예정 |
| 그래프/차트 구현 | ⏳ 예정 |

## 프로젝트 구조

```
dashboard/
├── app.py              # Streamlit 메인 앱
├── components/         # UI 컴포넌트
├── utils/              # 유틸리티 함수
├── config/             # 설정
└── requirements.txt
```

## 백엔드 API 연동

| 엔드포인트 | 용도 |
|-----------|------|
| `GET /stats` | 실시간 통계 (총 검사수, 불량률 등) |
| `GET /latest` | 최근 검사 이력 |
| `GET /defects` | 결함 타입별 집계 |
| `GET /images/defects/{filename}` | 결함 이미지 로드 |

## 주요 기능

### 1. 실시간 통계 대시보드
- 총 검사 수, 정상/불량 개수
- 불량률 (%)
- 평균 FPS

### 2. 최근 불량 정보
- 마지막 불량 이미지
- 결함 타입, 신뢰도, 위치

### 3. 검사 이력
- 최근 검사 목록 (테이블)
- 필터링/검색

### 4. 그래프
- 시간별 불량률 추이
- 결함 타입별 분포

## 서버 실행 방법

```bash
cd dashboard
pip install -r requirements.txt
streamlit run app.py
```
